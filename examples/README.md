# Examples

This directory contains example scripts demonstrating how to use the `headless_sidewalkreator` library.

## Available Examples

### 1. Full Pipeline Example (`01_full_pipeline_example.py`)

This example demonstrates the complete sidewalk generation pipeline:
- **Input**: Uses the carefully selected test polygon from `data_assets/test_data/polygon_4326.geojson` (a ~1.2 km² area in Curitiba, Brazil with good OSM data coverage)
- **Process**: Downloads OSM data automatically and generates sidewalks, crossings, and kerbs
- **Output**: Saves results to `examples/outputs/full_pipeline/`

**Note**: The test area was carefully selected for its good OSM data quality. OSM data download may take a few minutes due to the area size (~1.2 km²).

**What it generates:**
- `sidewalks.geojson` - Sidewalk line geometries with OSM tags
- `crossings.geojson` - Crossing line geometries connecting sidewalks
- `kerbs.geojson` - Kerb point geometries at crossing endpoints
- `merged_for_josm.geojson` - Combined file ready for JOSM/OSM upload
- `auxiliary/protoblocks.geojson` - Intermediate city blocks for analysis
- `auxiliary/pois.geojson` - Extracted points of interest
- `auxiliary/input_polygon.geojson` - Copy of input area for reference

**To run:**
```bash
python examples/01_full_pipeline_example.py
```

### 2. Protoblock Generation Example (`02_protoblock_generation_example.py`)

This example shows how to generate protoblocks (city blocks) using three different input methods:
- **Example 1**: From a polygon file (GeoJSON) - uses the carefully selected test data from Curitiba, Brazil
- **Example 2**: From a place name (geocoded) - e.g., "MIT, Cambridge, MA, USA"
- **Example 3**: From a bounding box (coordinates) - rectangular area

Protoblocks are the enclosed polygon areas formed by the road network - useful for urban analysis, block-level studies, and as input to other spatial algorithms.

**Output**: Saves results to `examples/outputs/protoblocks/`

**To run:**
```bash
python examples/02_protoblock_generation_example.py
```

Note: Examples 2 and 3 require internet access to download OSM data. Example 1 uses the local test polygon which also requires internet for OSM data download.

## Understanding the Outputs

### Sidewalks
Generated sidewalk segments along roads with the following OSM tags:
- `highway=footway`
- `footway=sidewalk`

### Crossings
Pedestrian crossings connecting sidewalks across streets:
- `highway=footway`
- `footway=crossing`

### Kerbs
Point features marking where sidewalks meet crossings:
- `barrier=kerb`

### Protoblocks
Intermediate polygons representing city blocks formed by the street network. These are useful for:
- Understanding the algorithm's intermediate steps
- Urban analysis and block-level metrics
- Filtering/refining sidewalk generation

## Viewing the Results

You can view the generated GeoJSON files in:
- **QGIS**: Free and open-source GIS software
- **JOSM**: OpenStreetMap editor (use the `merged_for_josm.geojson` file)
- **Any GIS software** that supports GeoJSON format

## Modifying the Examples

All examples are designed to be educational and hackable:
- Parameters can be adjusted in the `parameters={}` dictionary
- Different input areas can be used by changing the input source
- Output paths can be customized by modifying the `output_dir` variables

Feel free to experiment with different parameters and areas to understand how they affect the generated sidewalks!

## Troubleshooting

### OSM Download Timeout
If you encounter timeout errors when downloading OSM data:
- The examples already use a 180-second (3 minute) timeout
- For very large areas or slow connections, you can increase it further in the code
- The test polygon area (~1.2 km²) may take several minutes to download depending on network conditions

### Empty Results
If no sidewalks are generated:
- The area might already have complete sidewalk coverage in OSM
- Try using `ignore_existing=True` to generate all possible sidewalks
- Check that the input area contains roads

### Memory Issues
For very large areas:
- Break the area into smaller chunks
- Increase available memory
- Consider using a more powerful machine
